# BLUESHARK-STARK MEMORIZE PROTOCOL

## Purpose
Save valuable session learnings to BlueShark's brain for future sessions.

---

## MEMORIZE SEQUENCE

### 1. Update Current Status
Update `.brain/status-current.md` with:
- What was worked on today
- What was completed
- What is in progress
- Any blockers or issues
- Next session priorities

### 2. Log New Decisions (if any)
If any significant technical decisions were made, add to `.brain/decisions-log.md`:
- Decision title
- Date
- What was decided
- Rationale (why this approach)
- Impact on codebase

### 3. Update Workflow Patterns (if new patterns established)
If new UI/UX or code patterns were established, add to `.brain/workflow-patterns.md`:
- Pattern name
- Code example
- Key properties
- Where to use it

### 4. Create Session Log (for major sessions)
For significant sessions, create a session log:
`.brain/session-logs/YYYY-MM-DD-brief-description.md`

Session log template:
```markdown
# BlueShark Session Log - [Date]

## Overview
[1-2 sentence summary]

## Changes Made
### Feature/Fix 1
- What was done
- Files modified
- Key implementation details

### Feature/Fix 2
...

## Decisions Made
- [Any decisions and why]

## Testing
- [What was tested]
- [Test results]

## Next Steps
- [What should be done next]

## Notes
- [Anything else to remember]
```

---

## MEMORIZE CHECKLIST

Before ending session, ensure:
- [ ] status-current.md updated
- [ ] Any new decisions logged
- [ ] New patterns documented (if any)
- [ ] Session log created (for major work)

---

## After Memorize

Confirm to Sadin:
```
BLUESHARK BRAIN UPDATED

Session saved:
- Status: [brief status update]
- Decisions logged: [count or "none"]
- New patterns: [count or "none"]
- Session log: [created/not needed]

Ready for next session. Brain state preserved.
```
